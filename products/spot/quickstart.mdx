Place a first spot trade on the WhiteBIT API. The quickstart covers checking the Trade balance, placing a limit order on BTC_USDT, verifying the order, and canceling the order — all in under 5 minutes.

## Prerequisites

- A WhiteBIT account ([register](https://whitebit.com/auth/register))
- An API key with **Trading** permission ([create one](https://whitebit.com/settings/api))
- Funds in the **Trade** balance (transfer from Main balance if needed — see [Balances & Transfers](/concepts/balances))
- Familiarity with HMAC-SHA512 signing — see [Authentication](/api-reference/authentication)

<Warning>
  WhiteBIT has no public testnet or sandbox. All orders execute against the live orderbook.
  Use minimum order amounts and prices far from market to safely test order placement and cancellation.
</Warning>

<Steps>
  <Step title="Check Trade balance">
    Verify available funds before placing an order.

    <Tabs>
      <Tab title="cURL">
        ```bash
        # See /api-reference/authentication for signing details.
        # Compute BASE64_PAYLOAD from the JSON body and HMAC_SIGNATURE from the payload.
        curl -X POST https://api.whitebit.com/api/v4/trade-account/balance \
          -H "Content-Type: application/json" \
          -H "X-TXC-APIKEY: YOUR_API_KEY" \
          -H "X-TXC-PAYLOAD: BASE64_PAYLOAD" \
          -H "X-TXC-SIGNATURE: HMAC_SIGNATURE" \
          -d '{"ticker":"USDT","request":"/api/v4/trade-account/balance","nonce":"1700000000000"}'
        ```
      </Tab>
      <Tab title="Python">
        ```python
        import base64
        import hashlib
        import hmac
        import json
        import time
        import requests

        API_KEY = "YOUR_API_KEY"        # Replace with actual API key
        API_SECRET = "YOUR_API_SECRET"  # Replace with actual API secret
        BASE_URL = "https://api.whitebit.com"

        def send_request(path, data=None):
            if data is None:
                data = {}
            data["request"] = path
            data["nonce"] = str(int(time.time() * 1000))
            data_json = json.dumps(data)
            payload_b64 = base64.b64encode(data_json.encode()).decode()
            signature = hmac.new(
                API_SECRET.encode(), payload_b64.encode(), hashlib.sha512
            ).hexdigest()
            headers = {
                "Content-Type": "application/json",
                "X-TXC-APIKEY": API_KEY,
                "X-TXC-PAYLOAD": payload_b64,
                "X-TXC-SIGNATURE": signature,
            }
            return requests.post(BASE_URL + path, headers=headers, data=data_json).json()

        # Check USDT balance
        balance = send_request("/api/v4/trade-account/balance", {"ticker": "USDT"})
        print(balance)
        ```
      </Tab>
    </Tabs>

    **Expected response:**
    ```json
    {
      "USDT": {
        "available": "1000.00",
        "freeze": "0"
      }
    }
    ```

    `available` = funds ready for trading. `freeze` = funds locked in active orders.

    For Go and PHP examples, see [SDKs](/sdks).
  </Step>

  <Step title="Place a limit order">
    Create a limit buy order on BTC_USDT. Set the price well below the current market price to avoid immediate execution — the order remains open for safe testing.

    <Tabs>
      <Tab title="cURL">
        ```bash
        curl -X POST https://api.whitebit.com/api/v4/order/new \
          -H "Content-Type: application/json" \
          -H "X-TXC-APIKEY: YOUR_API_KEY" \
          -H "X-TXC-PAYLOAD: BASE64_PAYLOAD" \
          -H "X-TXC-SIGNATURE: HMAC_SIGNATURE" \
          -d '{"market":"BTC_USDT","side":"buy","amount":"0.0001","price":"30000","request":"/api/v4/order/new","nonce":"1700000000000"}'
        ```
      </Tab>
      <Tab title="Python">
        ```python
        order = send_request("/api/v4/order/new", {
            "market": "BTC_USDT",
            "side": "buy",
            "amount": "0.0001",
            "price": "30000"       # Price far below market — order will not fill
        })
        print(order)
        order_id = order.get("order_id")
        print(f"Order ID: {order_id}")
        ```
      </Tab>
    </Tabs>

    **Expected response:**
    ```json
    {
      "order_id": 12345678,
      "client_order_id": "",
      "market": "BTC_USDT",
      "side": "buy",
      "type": "limit",
      "timestamp": 1700000000.000,
      "amount": "0.0001",
      "price": "30000",
      "...": "..."
    }
    ```

    Request fields: `market` (trading pair), `side` ("buy" or "sell"), `amount` (quantity in base currency), `price` (limit price in quote currency). Optional: `client_order_id` (custom identifier), `stp` (self-trade prevention mode).
  </Step>

  <Step title="Check active orders">
    Verify the newly placed order appears in the active orders list.

    <Tabs>
      <Tab title="cURL">
        ```bash
        curl -X POST https://api.whitebit.com/api/v4/orders \
          -H "Content-Type: application/json" \
          -H "X-TXC-APIKEY: YOUR_API_KEY" \
          -H "X-TXC-PAYLOAD: BASE64_PAYLOAD" \
          -H "X-TXC-SIGNATURE: HMAC_SIGNATURE" \
          -d '{"market":"BTC_USDT","request":"/api/v4/orders","nonce":"1700000000000"}'
        ```
      </Tab>
      <Tab title="Python">
        ```python
        active_orders = send_request("/api/v4/orders", {"market": "BTC_USDT"})
        print(f"Active orders: {len(active_orders)}")
        for o in active_orders:
            print(f"  Order {o['order_id']}: {o['side']} {o['amount']} @ {o['price']}")
        ```
      </Tab>
    </Tabs>

    **Expected response (trimmed):**
    ```json
    [
      {
        "order_id": 12345678,
        "market": "BTC_USDT",
        "side": "buy",
        "amount": "0.0001",
        "price": "30000",
        "...": "..."
      }
    ]
    ```
  </Step>

  <Step title="Cancel the order">
    Cancel the limit order using the `order_id` returned in Step 2.

    <Tabs>
      <Tab title="cURL">
        ```bash
        curl -X POST https://api.whitebit.com/api/v4/order/cancel \
          -H "Content-Type: application/json" \
          -H "X-TXC-APIKEY: YOUR_API_KEY" \
          -H "X-TXC-PAYLOAD: BASE64_PAYLOAD" \
          -H "X-TXC-SIGNATURE: HMAC_SIGNATURE" \
          -d '{"market":"BTC_USDT","order_id":12345678,"request":"/api/v4/order/cancel","nonce":"1700000000000"}'
        ```
      </Tab>
      <Tab title="Python">
        ```python
        cancel = send_request("/api/v4/order/cancel", {
            "market": "BTC_USDT",
            "order_id": order_id
        })
        print(cancel)
        ```
      </Tab>
    </Tabs>

    **Expected response:**
    ```json
    {
      "order_id": 12345678,
      "market": "BTC_USDT",
      "side": "buy",
      "amount": "0.0001",
      "price": "30000",
      "...": "..."
    }
    ```

    The response confirms the canceled order details. The previously frozen funds return to the available Trade balance.
  </Step>

  <Step title="Check order history">
    Verify the canceled order appears in the execution history.

    <Tabs>
      <Tab title="cURL">
        ```bash
        curl -X POST https://api.whitebit.com/api/v4/trade-account/order/history \
          -H "Content-Type: application/json" \
          -H "X-TXC-APIKEY: YOUR_API_KEY" \
          -H "X-TXC-PAYLOAD: BASE64_PAYLOAD" \
          -H "X-TXC-SIGNATURE: HMAC_SIGNATURE" \
          -d '{"market":"BTC_USDT","request":"/api/v4/trade-account/order/history","nonce":"1700000000000"}'
        ```
      </Tab>
      <Tab title="Python">
        ```python
        history = send_request("/api/v4/trade-account/order/history", {
            "market": "BTC_USDT"
        })
        print(f"Order history entries: {len(history.get('BTC_USDT', []))}")
        ```
      </Tab>
    </Tabs>

    **Expected response (trimmed):**
    ```json
    {
      "BTC_USDT": [
        {
          "id": 12345678,
          "side": "buy",
          "amount": "0.0001",
          "price": "30000",
          "type": "limit",
          "...": "..."
        }
      ]
    }
    ```
  </Step>
</Steps>

All five steps completed successfully: balance checked, limit order placed, verified in active orders, canceled, and confirmed in history. The Trade balance reflects the returned funds.

<Accordion title="What just happened?">
  **HMAC-SHA512 signing** — Every private request includes three headers: `X-TXC-APIKEY` (your key ID), `X-TXC-PAYLOAD` (the request body Base64-encoded), and `X-TXC-SIGNATURE` (HMAC-SHA512 of the payload signed with your secret). This proves request authenticity without transmitting the secret over the wire. The `nonce` field in the body is a Unix millisecond timestamp that prevents replay attacks — the API rejects any request whose nonce is not strictly greater than the last accepted nonce for that key.

  **Trade balance vs Main balance** — WhiteBIT separates funds into two accounts. Main balance holds assets for deposits, withdrawals, and lending. Trade balance is the isolated pool used exclusively for spot order placement. Funds do not move between them automatically — an explicit transfer is required. If the Trade balance has insufficient funds when placing an order, the request fails; the Main balance is never drawn from.

  **freeze field** — When a limit order is accepted, the funds required for that order move from `available` to `freeze` in the Trade balance. They remain frozen until the order fills or is canceled. Step 3 confirmed the order was live with the balance frozen; after cancellation in Step 4, `freeze` returned to zero and `available` was restored — confirmed in Step 5.
</Accordion>
