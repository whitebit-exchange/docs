The Address Checker API enables automated cryptocurrency address screening for B2B accounts. The API evaluates a cryptocurrency address before a blockchain transaction is processed and returns a risk classification the partner can use to permit or block the transfer.

For general API conventions (base URL, error format, signing), see the [API Reference Overview](/api-reference/overview).

---

## Screening flow

<Steps>
  <Step title="Initiate the check">
    Submit a screening request to [Create address check](/api-reference/aml/create-address-check) with a supported `asset`, the `address`, and optionally a `network`. The endpoint returns a unique `id`.
  </Step>
  <Step title="Poll for the result">
    Call [Get check result](/api-reference/aml/get-check-result) with the returned `id`. Processing typically completes in under 60 seconds; the maximum is 3 minutes.
  </Step>
  <Step title="Act on the assessment">
    Once `status` becomes `Success`, the response contains `riskLevel` (`low` / `medium` / `high`) and `riskDescription`. Apply the partner's risk policy to decide whether to proceed with the transfer.
  </Step>
</Steps>

Use [List supported assets](/api-reference/aml/list-supported-assets) to discover the asset and network combinations the Address Checker accepts.

## Billing

<Warning>
Enabling the Address Checker endpoints explicitly authorizes WhiteBIT to charge **2 USDT** for each successful address check. The amount is deducted from the main balance when the check returns `status: Success`.
</Warning>

- Failed and invalid requests are not charged.
- Ensure sufficient main balance before initiating checks. A check that cannot be billed returns an `Insufficient balance` validation error and is not created.
- WhiteBIT notifies the account in advance of any pricing change. To maintain access, the account must accept the updated pricing terms after the notification. Until accepted, the system suspends Address Checker access.

## Risk gradation

The following descriptions are returned by the API and are not modified.

| Risk level | Description |
|------------|-------------|
| `low` | The address is verified as secure and suitable for fund transfers. |
| `medium` | The address reflects certain factors that warrant additional attention. Proceed with caution. |
| `high` | The address displays significant risk indicators, and the transaction will not be processed. Sending funds from this address is prohibited. |

---

## Check statuses

| Status | Description |
|--------|-------------|
| `In progress` | The system is processing the check. Processing may take up to 3 minutes. |
| `Success` | The system completed the check and provides the address screening result. |
| `Failed` | The system did not complete the address evaluation and did not charge the account. Retry later or contact Support. |

---

## Authentication & permissions

All Address Checker endpoints require an API key with the **Address Checker** permission enabled. See [Authentication](/api-reference/authentication) for the full HMAC-SHA512 signing process.

If the key lacks the permission, the endpoint returns `401` with `code: 4` (`API_KEY_ENDPOINT_RESTRICT`). If the account does not include access to the Address Checker API, the endpoint returns `404` — contact Support to request access.
