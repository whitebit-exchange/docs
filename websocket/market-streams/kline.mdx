## Query candlesticks (one-time)

Fetch historical candlestick data for a time range.

**Request**

```json
{
  "id": 1,
  "method": "candles_request",
  "params": ["BTC_USDT", 1738166400, 1738252800, 3600]
}
```

`params`: `[market, start_time, end_time, interval]`. Times are Unix timestamps.

**Response**

```json
{
  "id": 1,
  "result": [
    [1738166400, "104500.00", "104800.00", "105000.00", "104300.00", "8.12", "849601.23", "BTC_USDT"]
  ],
  "error": null
}
```

Each record is a positional array in the same format as the subscription update.

## Subscribe to candlesticks

<Steps>
  <Step title="Send subscription request">
    `params[0]` is the market. `params[1]` is the interval in seconds.

    ```json
    {
      "id": 1,
      "method": "candles_subscribe",
      "params": ["BTC_USDT", 3600]
    }
    ```
  </Step>
  <Step title="Receive confirmation">
    ```json
    {
      "id": 1,
      "result": { "status": "success" },
      "error": null
    }
    ```
  </Step>
  <Step title="Receive real-time updates">
    The server pushes `candles_update` every 0.5 seconds with the current candle. The update is a **positional array**.

    ```json
    {
      "id": null,
      "method": "candles_update",
      "params": [
        1738252800,
        "104800.00",
        "105200.00",
        "105500.00",
        "104700.00",
        "12.34567",
        "1296543.21",
        "BTC_USDT"
      ]
    }
    ```

    | Index | Field | Description |
    |-------|-------|-------------|
    | `[0]` | time | Candle open time (Unix timestamp) |
    | `[1]` | open | Open price |
    | `[2]` | close | Close price (current price for live candle) |
    | `[3]` | high | Highest price in the interval |
    | `[4]` | low | Lowest price in the interval |
    | `[5]` | volume | Volume in [stock](/glossary#stock) (base currency) |
    | `[6]` | deal | Volume in [money](/glossary#money) (quote currency) |
    | `[7]` | market | Market name |
  </Step>
  <Step title="Unsubscribe">
    ```json
    {
      "id": 2,
      "method": "candles_unsubscribe",
      "params": []
    }
    ```
  </Step>
</Steps>

## Interval constraints

The interval must meet specific alignment rules to ensure candles align with natural time boundaries.

<Accordion title="Less than 60 seconds">
  60 must be divisible by the interval.

  Valid: `1`, `2`, `3`, `4`, `5`, `6`, `10`, `12`, `15`, `20`, `30`

  Invalid: `7`, `11`, `13`, `17`
</Accordion>

<Accordion title="60 seconds to 1 hour (60–3599s)">
  Interval must be divisible by 60, and 3600 must be divisible by the interval.

  Valid: `60` (1m), `120` (2m), `180` (3m), `300` (5m), `900` (15m), `1800` (30m)

  Invalid: `420` (7m), `660` (11m)
</Accordion>

<Accordion title="1 hour to 1 day (3600–86399s)">
  Interval must be divisible by 3600, and 86400 must be divisible by the interval.

  Valid: `3600` (1h), `7200` (2h), `14400` (4h), `21600` (6h), `43200` (12h)

  Invalid: `18000` (5h), `25200` (7h)
</Accordion>

<Accordion title="1 day and above">
  Valid: `86400` (1d), `172800` (2d), `259200` (3d), `604800` (1w), `2592000` (1mo)
</Accordion>

<Note>
The constraints above ensure candles align with natural time boundaries (minutes, hours, days) for consistent charting.
</Note>
