## Implementation Guide

<Steps>
  <Step title="Initial Setup">
    1. [Contact the institutional sales team](#contact-options)
    2. Complete KYB verification
    3. Sign service agreement
    4. Provide technical requirements
    5. Schedule implementation
  </Step>
  <Step title="AWS Configuration">
    #### Network Setup
    - Network: 10Gbit connectivity minimum
    - Compute: 4+ vCPU cores minimum
    - Instance type: dedicated EC2 instances (recommended — low-performance VPS results in higher latency)
    - Security: IP address pool required for allowlisting

    #### Region Selection
    - **Europe (Frankfurt)** — recommended for EEA and EMEA trading operations
    - **Asia (Tokyo)** — recommended for APAC trading operations
    - Contact the account manager for connection details including base URLs and availability zone placement
  </Step>
  <Step title="Technical Integration">
    The technical team will assist with:
    - Network configuration including aws cloud latency optimized proxy
    - Performance testing
    - Trading system integration
  </Step>
</Steps>

## Available Connectivity

Colocation supports both REST API and WebSocket connections. The REST API provides access to a subset of endpoints optimized for low-latency trading. WebSocket connections enable real-time market data and order updates.

For the full list of 34 available colocation endpoints, quoting strategies, and infrastructure best practices, see the [Market Maker Integration Guide](/guides/market-maker-guide).

<Warning>
  Colocation provides a **subset** of the full WhiteBIT API. Only specific trading, collateral, and utility endpoints are available via the colocation infrastructure. See the [Market Maker Integration Guide](/guides/market-maker-guide#available-colocation-endpoints) for the complete endpoint list.
</Warning>
